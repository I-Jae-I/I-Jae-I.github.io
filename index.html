Project Idea

Overview

The project is to create a powerful IOS app that is able pull data from certain websites, mostly one that host e-books, light novels, manga, comics and a plethora more. Main motivation to create such app is due to the absence of such app that can do the required functions and features present in the app store now, though there are some apps that are able to do some of the features, however they all contain flaws in certain ways.

This app is mostly intended for IOS devices as a less powerful app currently exist for Android devices and the IOS apps that offer all listed services all have their own individual apps, requiring a combination of multiple apps whereas this will both eliminate the plethora of apps needed and the chunkiness that they present having to switch between multiple apps.

Description

This project app allows you to read any type of content from your IOS device with ease be it manga, light novels, e-books etc. from any source you know and prefer thus possessing a rather extensive range of catalogue and languages you speak/prefer (titles might be limited in other languages due to lack of translation).

Pre-existing sources from known sites will be available to choose from though if a title/series is missing or unavailable from the existing ones, a feature enables the user to add the source that the series/title is from allowing the user the flexibility and freedom to the desired content with little hassle.

Some of the features of the app are the ability to change the preferred page sizing/reading, type of page scrolling like infinite page scrolling, classic book animation page turning and regular left or right page turning. Other features include the ability to change the theme of the app from light to dark with intentions to allow the user to create and incorporate their own custom themes, and a community theme page that allows users to share their created themes; change font sizes and styles from a library or user custom styles; change the layout of the app by hiding some tabs or shifting how their content is displayed on the app; Custom update notifications that allows the user to decide how frequent the app scans for updated content; Custom content notification within the app which indicated how many new chapters or articles have been added to the title/series; and plethora more features.

Tools and Technologies and Skills Required

Since the application is targeted towards IOS devices the main program and language used to create the application will be XCode as the program and Swift as the language, also learning to use the Mac interface and the Swift coding language is required to begin as this language is different to other languages used to create applications for mobile devices.

An Apple computer and device will be required to create the app and test it, as currently XCode is an apple proprietary program which is the only program to my knowledge which allows the creation of apple apps as the code is different to that of android apps and the language used to create android apps. This requires the me to either have an Apple computer to create the app or purchase one, however there are ways that allows a programmer to use a windows machine to create and code IOS apps without an Apple computer, though this is ill advised from Apple.

Outcome

With the IOS app development successful then the need of multiple apps and the lack of content from existing sites will no longer be needed, as this app alone is powerful enough to replace existing apps from the same category from the app store and contains enough features and flexibility to do so. This also allows the user to fully customise the app to their preferred choice allowing a far emissive and enjoyable experience to the end user.
